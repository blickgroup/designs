Design notes — Ideas Bank: History filter (v1)

What this adds

A toggle on the existing Ideas Bank to flip between Active (today's default — un-allocated ideas, votable) and History (previously banked ideas that have been allocated to a plan/cycle, not votable).

What this does NOT add

  • No change to the active ideas bank — card grid, goal/effort/impact badges, vote button, submitter + time all stay as-is
  • No new status chip on active cards — being in the bank means approved; highlighting it is noise
  • Allocated ideas still drop out of the default view — bank stays clean

History-card differences (only applied when view toggle = History)

  • Plan strip at top — orange-soft band showing "CYCLE 2 · PLAN NAME" + a text link back to the cycle plan
  • Status chip in the badges row — In Planning / In Execution / Done / Parked — colour-coded
  • Vote button replaced with a muted "9 votes" readout — historical record, not actionable
  • View project CTA (black) takes the user to the full-page project workspace (stages 2–4 in the other mockups)
  • Slightly muted card background (#fafaf9) to visually separate from active cards if both were ever shown together

Plan/cycle filter

In History view only, a dropdown narrows by cycle (All cycles / Cycle 1 / Cycle 2 / …). Goal filter chips still apply on top.

Tradeoffs considered

  • Single unified view with status chips on everything — rejected. Muddies the active bank, invites voting on allocated ideas, and clutters the default mode with information nobody needs day-to-day.
  • Separate "Project Archive" nav item — rejected. Discovery problem: people looking for a banked idea will look in the bank first. A filter is where they'd expect it.
  • Status chip only, no plan strip — rejected. The plan/cycle context is the primary reason someone looks at history ("what did Cycle 1 actually ship?"). Surfacing it in the card saves a click.

Open questions for Shaun

  • Should History show all allocated ideas including Parked and Rejected, or split Parked/Rejected into a separate filter?
  • Within History, should cards group by cycle (section headers) or stay as a flat grid with the plan strip carrying context?
  • Default sort in History — most recent cycle first, or alphabetical?
