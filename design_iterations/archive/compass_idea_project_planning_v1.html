Design notes — Stage 2: Planning (empty/setup state)

What this shows

An idea has been approved and promoted to a project. The full-page workspace opens in setup mode: brief carried over from the idea submission, empty scaffolding for tasks/owner/docs, and a kickoff checklist. Design philosophy: scaffolding should feel inviting, not shouty. Dashed-border empty cards read as "here's what goes here" without chastising the user.

Nav pattern choice for this stage

This stage uses top tabs (Overview / Tasks / Follow-ups / Docs / Activity). Stage 3 (execution) is produced in both tabs and left-nav variants for comparison; this stage and stages 1 and 4 demo tabs for consistency. Rationale: top tabs sit naturally with the orange section-tag language Compass already uses, collapse cleanly on mobile, and the page isn't nav-heavy — left rail would be wasted real estate.

Two-column skeleton

Left column (~1fr) = main work area. Right column (320px) = context rail: status, owner, cycle, key dates, linked docs count, quick actions. Same pattern as TicketDetailPage.tsx (flagged by ux-researcher). Rail stays sticky on long scroll.

Tradeoffs considered

  • Wizard/step flow for setup — rejected. Fields should be fill-when-known, not forced-in-order. The kickoff checklist gives progress feedback without blocking.
  • Auto-generate tasks from brief via Nexus — flagged as open question (not shown). If we do this, the "Add first task" empty card becomes "Draft tasks from brief" + "Add task manually".
  • Hide side rail until setup complete — rejected. Seeing the shape of the populated page early builds a mental model; "Not set" with inline "Add" CTAs double as setup entry points.

Responsive behaviour

Below 960px: side rail drops under main content as a two-column card grid (Status/Owner/Cycle/Docs). Below 640px: all stacks vertical, tabs become horizontal scroll. Field users rarely hit setup — this is predominantly a desk surface.

Open questions for Shaun

  • Do we auto-draft first tasks from the idea brief via Nexus, or always start blank?
  • Does promoting the idea also auto-assign the submitter as owner, or require explicit owner assignment?
  • "Kickoff checklist" — useful nudge or paternalistic? Ditch it?

Brief — carried from idea

Problem. AU geothermal operators (Todd Energy, Origin) increasingly specify low-toxicity mud systems. Our current Blick fluid range is strong on cost/performance but has gaps on environmental classification and biodegradability data.

Proposal. Package an eco-branded variant of our existing high-temp fluid line with refreshed SDS, a biodegradability test from the AU lab partner, and a PDS+COA pack aligned to operator tender templates. Target the Todd Superhot tender (Q3) as the launch account.

Why now. Tender window closes Sep 2026. Tim has spoken with Todd's procurement lead. Blick's 2026 AU plan flags geothermal as a priority segment.
