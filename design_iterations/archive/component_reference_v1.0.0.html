📋 Component Reference Guide

Dark Theme (Left Panel)

  • Current implementation with hardcoded Tailwind classes
  • Uses classes like bg-brand-charcoal
  • Colors defined in Tailwind config
  • After migration: will use CSS variables instead

Light Theme (Right Panel)

  • Future state after CSS variable migration
  • Uses variables like var(--bg-primary)
  • Same visual appearance, different implementation
  • Enables seamless theme switching

✅ CORRECTED COLOR SCHEME: This reference now shows the proper brand-consistent color usage.

  • ✅ Orange (#f05323) - Primary actions, focus rings, section headings (brand color)
  • ✅ Gray - Secondary buttons, borders, structure (neutral hierarchy)
  • ❌ Cyan removed from light mode - Fixes WCAG accessibility failures (2.6:1 contrast)
  • ⚠️ Cyan in dark mode only - Optional for product code badges (8.2:1 contrast)

Migration Impact: Zero visual disruption in dark mode. Light mode now WCAG compliant with proper orange/gray hierarchy.
