Design Rationale
Title first — primary scan target
The task name is the only thing that uniquely identifies a card. Reading it first lets the eye scan the board by content, not by category.
Goal tag — secondary context
Helps filter by strategic initiative when needed, but doesn't need to precede the title. Colour coding (amber/blue/green) carries the signal.
Effort chip — sizing at a glance
S/M/L sits next to goal tag. Compact pill, muted colour — readable but not competing with the goal tag.
Cycle + due — merged, muted metadata
"C2 · May 28" consolidates two low-priority fields into one quiet element. Cycle number is context, not the primary signal — demoting it reduces visual noise without losing the data.
Assignee — left-anchored, always visible
Avatar + name chip in the footer left. Consistent position makes it scannable across all cards without reading the label.
