Design Tradeoffs
Option A — Kanban Option B — Swimlanes Option C — Table
Best for Drag status updates Cycle health overview Scanning + sorting
Shows titles ✓ On cards ✓ Inline rows ✓ Dedicated column
Cycle context Chip on card Primary structure Grouped rows
Progress visibility Good Best (per-cycle) Good
Status clarity Best (visual columns) Good (cells) Good (dot + row color)
Density Low (cards) Medium Highest (rows)
Mobile Poor (4 cols) Poor (dense grid) Best (rows scroll)
Sorting / filtering Not natural Limited Native
Build effort Low (exists) Medium Medium
