Design notes — Stage 4: Done / Parked (v2, historical record)

What this shows

Project completed (or parked). Read-only feel. Full journey is visible — submitted → banked → allocated to cycle → planning → execution → done. Outcomes retro block is prominent (the thing you want to remember next time). Task list is collapsed into a roll-up, all done or parked, greyed down. Activity feed becomes a closing timeline.

v2 change from v1: journey strip re-labeled to reflect the real lifecycle — an idea sits in the bank (approved, un-allocated) before being allocated to a 90-day cycle. Allocation is what creates the project.

Layout choices

  • Archive banner at top — immediately signals "this is historical" so no-one tries to add a task expecting action.
  • Stage badge flips from orange → green. Preserves the orange section-tag language for active work, green for closed. Matches the success-green throughout Compass.
  • Four-step journey strip replaces live meta cards near the top — gives the full story at a glance, with durations.
  • Retro / outcomes block becomes the hero on this page. Above the task list. What worked, what didn't, what we'd do differently. This is the compounding value — next cycle's planning phase should pull from these notes.
  • Side rail flips from "Quick actions" to "Summary metrics" (duration, tasks completed, slip days, final effort vs estimate).

Parked vs Done — handled the same way

A parked project uses the same layout but the stage badge goes grey, the retro becomes "Why we parked / when to revisit", and parked tasks show a ↓ marker instead of ✓. One template, two end-states. Avoids building two dead-end screens.

Still editable?

Tasks/follow-ups/docs are frozen (read-only). But the outcomes/retro block is editable for ~14 days after close (Shaun can still add learnings). Comments stay editable permanently — the project notes can be annotated forever. After 14 days a "Reopen project" CTA appears in the overflow menu if someone needs to revive it.

Tradeoffs considered

  • Auto-redirect to a separate archive view — rejected. Same URL, same layout, just a different mode. Linkable forever; less confusing.
  • Strip all the tabs — rejected. You still want to scroll through tasks historically; keeping tabs preserves the mental model from execution.
  • "Promote outcomes to vault" CTA — considered. Flagged for Shaun (open question).

Responsive behaviour

Journey strip becomes a vertical list below 720px. Retro stays prominent. Rail drops under.

Open questions for Shaun

  • Should closed projects auto-generate a vault note (Intelligence/projects/eco-fluid-au.md) so outcomes compound into Nexus search?
  • Retro edit window — 14 days enough? 30? Forever?
  • Should parked projects surface in a different pipeline view from Done, or mix them?

Outcomes · retrospective

What we shipped Delivered

  • Eco-certified SDS + PDS pack for HT-3000 Eco and HT-3500 Eco — signed off by Brent, reviewed by Shaun.
  • Tender response pack submitted to Todd Energy procurement (Sarah Mills), 12 Jun.
  • Biodegradability data from Aucklab — OECD 301B pass on both base fluids.
  • Cost model + margin scenarios built; eco premium holds at ~8% which Todd indicated is acceptable.

What slipped 18d over

  • Aucklab test scheduling cost us 12 days — they double-booked the rheology cell. Lesson: book lab slots first, not last.
  • PDS template rewrite took twice as long as Brent estimated because section 12 eco-tox data model had no precedent in Blick's system.
  • Shaun's sign-off waited 6 days in one round — need a clearer "waiting on" signal in-app.

What we'd do differently For C3+

  • Kick off lab partner scheduling on day 1, not week 4.
  • Create a shared "eco-tox SDS language" template in the vault so next product family can reuse it.
  • Wire follow-up "waiting reply" count into rail so reviewers' inbox depth is visible.
